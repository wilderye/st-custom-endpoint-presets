## A. 主面板基础功能（原有 + 重构）

### A1. 面板打开

- [ ] 通过扩展菜单（魔棒）点击"隐藏楼层管理"，面板弹出
- [ ] 面板显示正确的"最新消息层数"（= `SillyTavern.chat.length - 1`）
- [ ] 没有聊天打开时（`SillyTavern.chat.length === 0`），toast 提示"需要先打开一个聊天"，面板不打开

### A2. 已隐藏楼层展示

- [ ] 无隐藏楼层时，显示"无"
- [ ] 有隐藏楼层时，显示合并后的范围文本（如 `0-2, 5, 7-8`）
- [ ] 验证范围合并逻辑：输入 `[0,1,2,5,7,8]` → 输出 `0-2, 5, 7-8`
- [ ] 验证单条隐藏：输入 `[3]` → 输出 `3`
- [ ] 隐藏楼层判断逻辑：确认使用 `SillyTavern.chat[i].is_system === true` 而非旧的 `getChatMessages`

### A3. 隐藏/显示操作

- [ ] 起始 > 结束时，toast 警告
- [ ] 起始或结束为负数时，toast 警告
- [ ] 起始或结束不是数字（空、NaN）时，toast 警告
- [ ] 起始 = 结束时，正常执行（隐藏/显示单条）
- [ ] 隐藏/显示执行后，面板中的"已隐藏楼层"文本自动刷新
- [ ] 验证使用 `SillyTavern.executeSlashCommandsWithOptions` 而非旧的 `triggerSlash`

## B. 直接跳转

### B1. 输入校验

- [ ] 输入为空时，toast 提示
- [ ] 输入为非数字文本时，toast 提示
- [ ] 输入为负数时，toast 提示
- [ ] 输入超过最新楼层号（如 lastId=499，输入 500）时，toast 提示
- [ ] 输入为小数时（如 3.5），自动取整为 3（`Math.floor`）
- [ ] 输入为 0 时，正常执行（第一条消息）
- [ ] 输入等于 lastId 时，正常执行（最后一条消息）

### B2. 卡顿确认提示

- [ ] gap ≤ 100 时（如 firstId=400，输入 350），**不弹确认**，直接跳转
- [ ] gap > 100 时（如 firstId=400，输入 0），弹出确认框
- [ ] 确认框点"是"：执行 `executeSlashCommandsWithOptions('/chat-jump 0')`
- [ ] 确认框点"否"：不跳转，回到面板
- [ ] 目标楼层已在 DOM 中（如 firstId=400，输入 450）时，gap=0，直接跳转
- [ ] `document.querySelector('#chat .mes')` 返回 null 时，firstId 默认为 0，不弹确认

### B3. gap 计算的正确性

- [ ] gap = `Math.max(0, firstId - targetIndex)`，而不是 `chat.length - targetIndex`
- [ ] 当所有消息都已渲染（firstId = 0）时，gap 始终为 0，任何跳转都不弹确认

## C. 预览跳转弹窗

### C1. 弹窗打开

- [ ] 输入校验与直接跳转相同（B1 所有条目）
- [ ] 弹窗使用 `SillyTavern.callGenericPopup` + `POPUP_TYPE.DISPLAY` 弹出
- [ ] 弹窗启用 `allowVerticalScrolling: true`

### C2. 初始范围计算

- [ ] target=50, lastId=499 → 范围 [40, 60]，共 21 条
- [ ] target=5, lastId=499 → 范围 [0, 15]，不足前 10 条时取 0 为下界
- [ ] target=495, lastId=499 → 范围 [485, 499]，不足后 10 条时取 lastId 为上界
- [ ] target=0, lastId=499 → 范围 [0, 10]
- [ ] target=499, lastId=499 → 范围 [489, 499]
- [ ] target=3, lastId=5 → 范围 [0, 5]，聊天总数少于 21 时显示全部

### C3. 消息条目渲染

- [ ] 每条消息显示楼层号（如 `#40`）
- [ ] 每条消息显示正确的发送者名称（角色名 / 用户名）
- [ ] 发送者名称判断：`msg.is_user ? SillyTavern.name1 : msg.name`（角色可能有自定义名称）
- [ ] 消息内容使用 `SillyTavern.messageFormatting()` 渲染
- [ ] `messageFormatting` 的参数传递正确：`(msg.mes, senderName, is_system_flag, msg.is_user, mesid)`
- [ ] 非中心楼层：默认折叠（CSS `max-height` 限制），可见"展开"按钮
- [ ] 中心楼层（目标楼层）：默认展开，有高亮样式
- [ ] 点击"展开"后，消息完整显示，按钮文本变为"折叠"
- [ ] 点击"折叠"后，消息恢复折叠状态

### C4. 跳转图标按钮

- [ ] 每条消息右侧有跳转图标按钮
- [ ] gap ≤ 100 时：直接关闭预览弹窗，执行跳转
- [ ] gap > 100 时：**先弹确认框**，确认"是"后再关闭预览弹窗并跳转
- [ ] 确认框点"否"时：不关闭预览弹窗，不跳转，用户留在预览页面

### C5. 重新预览

- [ ] 顶部有一个目标楼层输入框，初始值 = 进入预览时的目标楼层
- [ ] 修改输入框值后点击"预览"，弹窗内消息列表以新目标为中心 ±10 重新渲染
- [ ] 输入为空/非数字/超出范围时，toast 警告
- [ ] 输入小数时自动取整（`Math.floor`）
- [ ] 点击"显示更多"**不更新**该输入框的值
- [ ] 用户手动修改输入框值后，该值保持用户填写的内容，不被其他操作覆盖

### C6. 显示更多楼层按钮

- [ ] **顶部按钮** `[显示更多楼层 ↑]`：
  - [ ] 点击后范围 start 减 10（如 40→30）
  - [ ] 到 0 时，按钮隐藏
  - [ ] 当前 start 已经是 0 时，初始就不显示该按钮
- [ ] **底部按钮** `[显示更多楼层 ↓]`：
  - [ ] 点击后范围 end 加 10（如 60→70）
  - [ ] 到 lastId 时，按钮隐藏
  - [ ] 当前 end 已经是 lastId 时，初始就不显示该按钮
- [ ] 点击后消息列表重新渲染（清空 + 重建）
- [ ] 点击后**不自动滚动**，由用户自行操作
- [ ] 顶部目标楼层输入框的值**不变**

## E. 异常和边界

### E1. 数据边界

- [ ] 聊天只有 1 条消息时，隐藏/显示/跳转是否正常
- [ ] target = lastId 且 lastId = 0 时（只有一条），预览窗口应只显示 1 条
- [ ] 预览弹窗中目标楼层恰好是第 0 条或最后一条时，高亮正确

### E2. 弹窗冲突

- [ ] 预览弹窗中点击跳转 → 弹确认框 → 确认框显示在预览弹窗之上
- [ ] 确认"是"后，确认框和预览弹窗都关闭
- [ ] 确认"否"后，确认框关闭，预览弹窗保留

### E3. API 调用

- [ ] `SillyTavern.chat` 数组为空或 undefined 时的防御（A1 已覆盖）
- [ ] `SillyTavern.messageFormatting` 对特殊内容（含 HTML、含图片 URL、含代码块）的渲染正常
- [ ] `executeSlashCommandsWithOptions` 返回后检查 `isError` 字段
